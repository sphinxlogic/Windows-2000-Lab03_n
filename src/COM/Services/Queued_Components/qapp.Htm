Set the "Queued" property for the QCShip and QCNotify applications

In this step you will set the "Queued" property for:

    • QCShip
    • QCNotify
  1. If not already started, start the Component Services Explorer by selecting Start->Programs->Administrative Tools-> Component Services.
  2. Navigate to and highlight Console Root\Component Services Explorer\Computers\My Computer\COM+ Applications
  3. In the right pane you will see the icon for QCShip.
  4. Right click on QCShip and select Properties.
  5. Select the Queuing tab.
  6. Check the Queued option.
  7. Do not select Listen.
  8. Click OK.
  9. Repeat the step for QCNotify.

This step actually creates the underlying MSMQ queues that provide the queuing services for these queued components. You can see the effect of setting the Queued property on the COM+ application by running the MSMQ MMC Snap-in (see Important MSMQ Information) and looking in the Public Queues folder.
