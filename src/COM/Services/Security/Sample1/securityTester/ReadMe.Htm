Security Tester

Overview

VB application using Security Sample methods


To Build:

- Ensure that the Security Sample's VC, VB, & VJ Dlls are all built

- Open ServicesSecurityTester.vbp

- Ensure all references are resolved

- Make ServicesSecurityTester.exe


To Configure:

- In COM Services Explorer create a new package named: Security Tester

- Right Click on the New COM+ Application

- In Properties- Security Tab- Authorization: Check the "Enforce access checks for this application" checkbox

- Click "OK"

- Under the Security Tester COM+ Application, create this Role: Administrator (Beware of Case Sensativity)

- Add an intended User to Administrator role

- Install the VC, VB, & VJ Security Sample Dlls

- For Each Component: In Properties- Security Tab: Check the "Administrator" Checkbox


To Run:

- Run ServicesSecurityTester.exe

- Clicking the "Who is logged on?" buttons will bring up up a message box displaying the Answer

- Clicking the "Are you an Administrator?" buttons will tell you if you are listed in the "Administrator" Role of the Services_Security package

- Change Users in the Administrator Role or Security Options with the "Security Tester" Application to manipulate values returned by the security methods
