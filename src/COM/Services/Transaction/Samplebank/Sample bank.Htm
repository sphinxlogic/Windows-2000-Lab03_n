Sample Bank

Overview

The Sample Bank application consists of a simple VB form that allows a user to perform basic bank transaction such as debits, credits and transfers.  The VB performs these action but instantiating COM+ objects to make database updates using ADO.  These COM+ objects are implemented in three differenet programming languages and the UI allows the user to select which version or the component to use.

The components can run be installed as standard COM objects or as COM+ objects.  If used as standard COM objects, the components do not participate in transactions.  BankAdmin contains a sample script that uses the COMAdmin scripting components to create a COM+ application and register the components.

Build Notes

Build each of the components using its respective tool and then execute the InstallSampleBank.vbs to create the Sample Bank COM+ Application and register the components in COM+. (For the running of the VJ account component ensure that the CLASSPATH user environment variable has the path to the \Transaction\Samplebank\vjacct folder listed first)

Remarks

These samples require that a FILESDSN called BankSample is present and correctly configured.  This DSN should have been installed by the Platform SDK Installer.  If BankSample.dsn does not exist, create a FILEDSN that points at the pubs database on any valid SQL Server.

The subdirectories step1 through step8 in vbacct demonstrate one possible development path for the sample code.  The final version of the sample of the code is contained in vbacct directory itself..
