Known Issues

This file contains late-breaking information about issues you may encounter when building or using the Microsoft Exchange Server libraries and samples that accompany the Platform SDK in the Database and Messaging component.

Issues are organized under the following broad categories:

  • Library and COM Component Issues
    Issues related to the COM components or the ExchSDK.lib library.
  • Samples Issues
    Issues related to the samples themselves.
  • Miscellaneous Issues
    Any other issues.
The samples are listed in the following sections according the convention used in the documentation: the name of the sample matches the directory name in which the sample resides. This convention also applies to the library and components.

Library and COM Component Issues

Compiling COM Components using Microsoft Visual C++ 6.0

  • The new COM components ACL, Rule, and AcctMgmt are based on the Active Template Library (ATL). If you build one of the new COM components on one machine with the MinSize configuration, and then copy and register the component on a separate server machine, that machine will also require the new 3.0 version of the Active Template Library (ATL) dynamic link library ATL.DLL for the component to function properly. This run-time library is installed with the new Microsoft Visual Studio 6.0 release.

Building Administrative Extensions

  • If you build the administrative extensions using the Microsoft Foundation Classes (MFC) with the NDEBUG flag set, you must ensure that the MFC42.DLL, MFC42D.DLL, MSVCRT.DLL, and MSVCRTD.DLL dynamic link libraries are in the link path. If these files are not in the linker's path, a link error occurs.
  • If you build the administrative extensions using Visual C++ 6.0 and want to run the sample on a machine that does not have Visual C++ 6.0 installed, you must copy the MFC42.DLL, MFC42D.DLL and MSVCRTD.DLL libraries to the System32 directory on the machine running the sample.

Compiler Produces Warning Messages

  • The compiler can produce warning messages when building the samples. You can safely ignore these warnings.

AcctMgmt Component

  • This component does not check for invalid information supplied to create new accounts. Invalid information includes group names containing invalid characters, passwords longer than the maximum length, and usernames containing periods or spaces.
  • The ChangeOwnerOfSecDescriptor method does not check whether the passed in OldOwner parameter is part of the security descriptor.

ACLObject Component

  • If you are not the Exchange Server administrator, you must logon to a MAPI session before creating the ACL Object.

Rule Component

  • The Information Store accepts a maximum of 32K bytes of rule logic on a folder. This limit translates to approximately 200 simple rules on a folder.
  • The Rules.Item property must be explicitly cast to an integer when using scripting languages such as VBScript. For example, 
    myRule = objRules.Item(CInt(num))
  • The Actions.Add method always adds actions to the end of the collection.
  • The Action.Arg property does not accept Unicode characters. It instead substitutes the ? character in place of each Unicode character.
  • The SizeCondition.Size property erroneously allows negative values to be set.

Sample Issues

ACLASP Sample

  • Searching (using the Find button in the sample) the Recipients directory container using one character, such as 'n', 's', or 'x', can return unexpected results (depending on your set of installed proxy addresses), such as "SMTP", "X400", "NOTES", in the list of display names, aliases, and proxy addresses. This problem occurs during Ambiguous Name Resolution (ANR) searches using a CDO filter. To avoid this problem, submit a query using as many letters as possible.
  • Searches (using the Find button in the sample) of the Recipients directory container will not find members if their respective alias and display name both contain a leading space. For example, if you have a distribution list with alias and display name both equal to " Admins" (i.e. space - Admins), it will never appear as a result of a search.

AEditor Sample

  • The editor terminates abnormally when the Routing Wizard dynamic link library is not installed. To avoid this problem, install the Routing Wizard dynamic link library (Exrtobj.dll) from Microsoft Exchange Server 5.5 Service Pack 1.
  • If you use the Exchange Administrator program to delete a mailbox used by a profile that is in the list of previously viewed profiles, and then try to access the profile in the sample, an error with the message "Method 'Session' of object 'IEvents' failed" is generated.
  • The sample neither displays nor allows you to modify custom agents.
  • The sample does not check whether folder permissions are changed after setting the folder. If the permissions on a folder are changed you must restart the sample to pick up the changes.
  • The sample does not gracefully degrade when the environment changes radically, such as when the network is unreachable or the information store has stopped.
  • The sample allows concurrent access to a folder.
    If user A opens a folder to add a new agent, and before they save the new agent User B also opens the folder, both users can save their agent to the folder. The result is two agents available in the selected folder, with two different scripts. The map will be whichever was the last one saved.
    One possible solution this problem is to give only one user permission to create agents on a folder.
  • The sample erroneously allows you to create an agent on the Calendar folder.

DLUpdate Sample

  • This sample only searches the top-level Recipients container for entries. If you have multiple Recipients directory containers and you want to search them all, you must modify the sample code.
  • You can create an illegal directory name using a client that supports Unicode characters by entering them in the alias field. To avoid this problem, do not enter Unicode characters in the alias field.
  • The sample does not correctly display non-Roman characters in the name of a distribution list. Each non-Roman character erroneously appears as the '?' character .
  • In the file Constants.inc, the result-set size for ActiveX Data Objects (ADO) queries is set to 100 using the mcADS_RS_PAGE_SIZE constant. This matches the number supported for the default Exchange Server 5.5 LDAP configuration. If you increase this value in the sample to over 100, make sure to update the LDAP search configuration value on the Exchange Server to match or exceed the new value. If this is not done, the sample will generate an error with the message "The name limit for the local computer network adapter card was exceeded" if the size of the query result set is larger than the LDAP setting. The LDAP search result-set size can be modified throught the Exchange Adminstrator application: navigate to the property page located at Configuration, Protocols, LDAP (Directory) Site Defaults, Search, and update the "Maximum number of search results returned" to the desired value.
  • The sample generates an error if you change the search parameter while the sample is returning the query results. To avoid this problem, wait until the sample has completed refreshing the query results before changing the search parameter.

Gateway Sample

  • The sample does not accept data from a DAT file that was produced with the CFGCODE sample executable and a custom INI file.
  • If you want to build the binary, you must edit the INF file to ensure the file size and date entry for each file is correct.
  • The sample exits when it receives a MAPI_E_NETWORK_ERROR error when polling for new messages.
  • The sample can create an unusable address if the organization or site name contains a space.

Incremental Change Synchronization (ICS) Visual Studio Wizard Sample

  • Building with the MFC library as a statically linked library produces a C++ run-time library conflict while linking.
    To avoid this problem, always build with the MFC library as a shared DLL.
  • The compiler can produce warning messages when building the samples.  To disable the warnings, you can add certain libraries to the project's Link Ingore list.
    • Debug Configuration - You can safely ignore warnings about links to MSCVRT.LIB (Microsoft C runtime libraries) or add it to the Ignore list.  If you encounter link errors to the MSVCRTD.LIB (Debug version of the C runtime), make sure this library is in your link path as this library should be linked property for the debug configuration.
    • Release Configuration -  You can add LIBC.LIB and MSVCRTD.LIB to the Project, Settings, Link, Input, Ignore libraries field in Visual C++.
  • The ICS NT service fails to start after service is installed.
    To alleviate this problem, copy Folder_EIDS.sav from the source directory into the directory containing ICSAgentNTService.awx.

Instform Sample

  • This form installation sample is not supported on Alpha machines.

Mailbox Sample

  • You can create an illegal mailbox name using a client that accepts double-byte (Unicode) characters by entering double-byte characters in the alias field.
    To avoid this problem, do not enter double-byte characters in the alias field.
  • You can accidently try to add the same mailbox twice.
    If you click the Submit button twice on the Add Mailbox page, the sample reports a Recipient is already on file error. The sample does not report that a mailbox is successfully added after the first click.
  • You cannot access an Exchange server on a remote computer if NT challenge and response security is used for authentication.
  • The sample allows you to create an illegal NT account, such as an account with periods or spaces in the account name.
  • You can use this sample to only search the top-level container for entries. If you have multiple Recipients directory containers and you want the sample to search them all, you must modify the code.

RuleASP Sample

  • Searching (using the Find button in the sample) the Recipients directory container using one character, such as 'n', 's', or 'x', can return unexpected results (depending on your set of installed proxy addresses), such as "SMTP", "X400", "NOTES", in the list of display names, aliases, and proxy addresses. This problem occurs during Ambiguous Name Resolution (ANR) searches using a CDO filter. To avoid this problem, submit a query using as many letters as possible.
  • Searches (using the Find button in the sample) of the Recipients directory container will not find senders or recipients if their respective alias and display name both contain a leading space. For example, if you have a distribution list with alias and display name both equal to " Admins", it will never appear as a result of a search.
  • The "Delete" action for a rule deletes the message rather than moves the message to the "Deleted Items" folder. This is by design for the sample. If you want to change this behavior to move the message to the "Deleted Items" folder, change the action from delete to move, and set the Action.Arg property to the "Deleted Items" folder (see the Action class example).
  • The sample cannot search mailboxes with leading spaces.
  • The sample will not allow you to log on to a mailbox using a different character set than the one used to create the mailbox alias.
  • If a public folder does not have owner permissions, you cannot update more than one rule.
  • If you create a folder rule without a condition, the sample cannot display the rules on that folder.

TOOLFORM Sample

  • This sample is only supported for use with the Microsoft Exchange client. The form does not function properly with any version of Microsoft Outlook.

Miscellaneous Issues

RECIPIENTINFO Structure

  • If you do not supply a last name in the pszLastName member, the proxy generator uses the common name member, pszCommonName, not the first name member, pszFirstName. This can result in an erroneous proxy address.
