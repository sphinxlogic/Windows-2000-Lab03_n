----Languages supported by Windows Installer----
The table below lists the languages for which the Windows Installer has
localized resource strings, and for which this SDK has supplied localized
Error and ActionText tables. The associated ANSI codepage for each of the
languages is given, and unless a package or table is truly neutral, i.e.
having no extended characters in any of its strings, the database or 
table is tagged with this codepage. A LangId marked with an asterisk '*'
indicates that resource strings are stored in the installer as the base
languages (low-order 10 bits or LangId & 0x3FF), in order to allow use
with all dialects, e.g. US English, British English, Austrailian English.

----Steps in localization of an installer package----
The following is a suggest sequence to produce a localized package.
See the help topic "Code Page Handling" for more details. For use
on Windows 9x, which does not support Unicode, the system codepage
must match the target codepage for the localized installer package.
Note that database column types are designated as localizable when
appropriate, as an aid to development of localization tools.
1. Set the database codepage, unless starting with a neutral (ACP = 0) package.
   Use WiLangId.vbs or equivalent code, importing the _ForceCodepage table.
2. Import localized Error.idt and ActionTe.idt tables for the target language.
   Use WiImport.vbs or MsiDb.exe or use the MsiDatabaseImport API function.
3. Localize other data in localizable columns, using a table editor such as
   Orca.exe, or by using the installer database API SQL access functions.
4. Set the ProductLanguage property in the Property table to the target
   LangId, as part of the localization or by using WiLangId.vbs.
5. Set summary information stream property 7 (Template) to indicate the target
   LangId. Use WiLangId.vbs, WiSumInf.vbs, MsiInfo.exe, or consult MSI help.
6. Only if the text summary information stream strings are localized should the
   summary information property 1 (Codepage) be set to the associated codepage.
7. Commit the database changes by saving the package with the editing tool
   or by calling the MsiDatabaseCommit API function.
The summary information LangId should match the ProductLanguage property,
and the database must be set to the codepage that supports that language.
