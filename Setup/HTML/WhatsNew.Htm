What's New in this Release

The Microsoft® Platform Software Development Kit (SDK) now supports development for Microsoft® Windows NT® Server/Enterprise Edition, Microsoft® Windows NT 4.0 and Microsoft® Windows® 2000 Release Candidate 2 (RC2), Windows® 95, and Windows® 98, and Internet Explorer version 3.0 and later. These Windows products provide a wealth of new technologies that allow your application to reach new heights. Below are highlights of some of the new and updated technologies.

Win32 API
Custom Scripting for Remote Access Connections
Extensible Authentication Protocol (EAP)
Multicast Group Manager
Routing Table Manager Version 2

Win32 API

The definitions of the Win32 API elements have been updated with new Win64-compatible types.

For more information, see the Win64 Programming Preview in the Design Strategies and Standards node of the Platform SDK documentation.

Custom Scripting for Remote Access Connections

Remote Access Service (RAS) now supports custom-scripting DLLs. Custom-scripting DLLs run on the RAS client. A custom-scripting DLL can send and receive data over the port for the RAS connection. RAS provides the DLL with the phone-book entry that RAS dialed for the connection. As a result, the DLL can take "special-case" actions for particular phone-book entries.

For more information, see the RasCustomScriptExecute section in the Platform SDK documentation.

Extensible Authentication Protocol (EAP)

You can now use Extensible Authentication Protocols (EAPs) to authenticate multilink and callback connections. The PPP_EAP_INPUT structure and the RasEapGetIdentity functions have been modified to support the new functionality.

For more information, see the Multilink and Callback Connections section in the Platform SDK documentation.

Multicast Group Manager

The Multicast Group Manager (MGM) API makes it possible to access the multicast routing capabilities of Microsoft Windows 2000 Server. Using MGM, you can write routing protocols that interact with MGM to receive and forward multicast data using both calls to MGM and callbacks from MGM. You can also write administrative applications that track group membership.

For more information, see the Multicast Group Manager section in the Platform SDK documentation.

Routing Table Manager Version 2

Routing Table Manager Version 2 (RTMv2) is a new version of the Routing Table Manager API. RTMv2 makes it possible to use the routing table manager for Microsoft Windows 2000 Server, and to develop routing protocols for the Internet Protocol (IP) transport. Routing protocols can modify and query the routing table, and they can use protocol-specific information to determine interoperability actions. RTMv2 can also be used to develop methods that are exported for other routing protocols to invoke.

For more information, see the Routing Table Manager Version 2 section in the Platform SDK documentation.
