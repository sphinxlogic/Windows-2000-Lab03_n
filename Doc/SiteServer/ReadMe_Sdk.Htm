Site Server 3.0 SDK version 1.1 Readme

This readme contains important, late-breaking information about the Microsoft® Site Server SDK version 1.1. You can access updated versions of this SDK on the Web, a component of the Microsoft Platform SDK.

Known Issues

The following are known issues for this release:

General

Microsoft recommends that Java programmers use the latest Java SDK tools for accessing COM objects.

You must use Microsoft® Internet Explorer 4.x in order to read the documentation.

Content Deployment

  • Be careful when using the ReplicationEvent interface in Visual Basic to access the event parameter passed to SiteServerEventSink_OnEvent(). If your code accesses the ReplicationEvent.NumFiles property, it can cause an access violation when the Content Deployment service, Crssrv.exe, shuts down.
  • The required Active project properties are the same as those required for Push projects, with the following differences:
    • Name Description
    ReplicationMethod = "active" Set this property to "active" for Active projects.
    LocalDirectory = CAB_File This property is the (Unicode) name of the local CAB file. It cannot be empty ("").
    Flags = 0  
  • The required Metabase project properties are as follows (any other values are invalid):
    • Name Description
    ReplicationMethod = "SendInet" Set this property to "SendInet" for Metabase projects.
    LocalDirectory = WebSiteName This property is the name of the Web site to which the Metabase information is replicated.
    MetabaseReplication = Yes Set this property to Yes for Metabase projects.
    Flags = RF_FORCE_REPL OR
               RF_FASTMODE    		 This property consists of bitmapped values that you can OR together as needed.
  • The Include directory contains the following files that you can use when programming Content Deployment administrative tasks in various programming languages:
    • Name Contents
    Crsmsg.bas Status code definitions for Visual Basic
    Crsmsg.h Status code definitions for C/C++
    Crsmsg.inc Status code definitions for VBScript
    Crstypes.bas Type information for Visual Basic
    Crstypes.h Type information for C/C++
    Crstypes.inc Type information for VBScript

Personalization and Membership

The following information should be used in addition to the sample.doc found in the \Samples\Siteserver\PM\Docs directory. This information applies to the samples within the \Samples\Siteserver\Pm directory. Specific file names are referenced below.

The sample.doc file makes reference to "DN" and "CN". DN is the Distinguished Name which looks like "o=Microsoft,ou=Admin,cn=Schema". CN is the Common Name and looks like "cn=Schema".

The "server:host" for the local physical machine is localhost:1002 unless you have changed the port number.

SetAttribute.vbs
    The file format used is specified within the vbs file itself.
CancelUsers.vbs
    The attribute to cancel users is LASTVISIT.
ModifyAttribute.vbs
    Only the DisplayName,Description, and Constraint can be changed.
VerifPwd.vbs
    This sample only works if Forms Authentication is used. Once the name/password is copied from the NT user account to the MDM, any authenication model can be used. On a default installation, you should not have to change the user name or password in the file.
Logoff.asp
    This file should be used for Kiosk-type clients where there are many users. This example removes the GUID from the cookie file.
DumpACL.vbs
    You must have anonymous access checked on your LDAP server for this to work correctly.
