MS DTC Oracle Test Program

 

Introduction

 

Microsoft Transaction Server components may access both Oracle7 and Oracle8 database servers. They may access Oracle database servers on Windows NT, Unix, and other systems. They may access Oracle Workgroup Servers, Oracle Enterprise Servers, and Oracle Parallel Servers.

After installing and configuring Oracle support, you should validate your Oracle installation by running the Oracle test program. The Oracle test program determines whether you can connect to your Oracle database using Oracle's XA facility. The Oracle test program uses standard Oracle interfaces and transaction facilities in much the same way that MS DTC uses them. It makes no use of COM+ or Microsoft Distributed Transaction Coordinator. Therefore, failure of the test program indicates that your Oracle system is improperly installed or configured. If the Oracle test program fails, reinstall and reconfigure Oracle, or contact the Oracle Support Organization for assistance

We have provided you with both the source code and the executable file for the Oracle test program. You may simply run the Oracle test program as described below. Alternatively, you may compile the source code using the C++ compiler to produce the Oracle test program.

 

Running the Oracle Test Program

 

1.          Install and configure Oracle support as described in the instructions for using Oracle with COM+. These instructions are included with the documention describing how to administer MS DTC.

2.          Verify that you have installed all of the correct versions of the software as described in the instructions for using Oracle with COM+.

3.          Create an ODBC DSN that refers to your Oracle database. Ensure that your DSN uses the new Microsoft Oracle ODBC 2.0 driver or later.

4.          Ensure that you have enabled Oracle XA support as described in the instructions for using Oracle with COM+..

5.          Ensure that the Windows registry refers to the proper version of Oracle Client software. Use REGEDIT to examine and modify the values stored under the HKEY_LOCAL_MACHINE\SOFTWARE\MICROSOFT\MSDTC\MTxOCI registry key. Under this key there are two string-named values that specify the names of the Oracle Client software DLLs.

If you are using Oracle7.3 ensure that these keys refer to the Oracle7.3 Client software DLLs.

OracleXaLib " xa73.dll"

OracleSqlLib " sqllib18.dll"

If you are using Oracle8 ensure that these keys refer to the Oracle8 Client software DLLs.

OracleXaLib "xa80.dll"

OracleSqlLib "sqllib80.dll"

6.       Delete all existing Oracle trace files from the machine containing the COM+ components that access the Oracle database. The easiest way to do this is to use the Windows Explorer to locate and delete all *.TRC files. If the Oracle test program fails, the trace files may help you determine the source of the problem. By deleting all obsolete trace files, you make it easier to find any newly created ones.

7.          From the MS-DOS Command prompt run the Oracle test program (MsDtcOra.exe) and supply your Oracle server user ID, password, and service name. For example:

c:>MsDtcOra.exe  -U<user id> -P<Password> -S<Service_Name as contained in the TNS file>.

8.          If you run the test program with no parameters, it will display help information that describes how to run the program. The test program will display information about each Oracle operation it performs and will indicate whether each operation was successful.

9.          If the Oracle test program is able to connect to your Oracle database server without error, then it is very likely that MS DTC will work with Oracle also. If the Oracle test program reports any errors, follow these steps:

9.1.      Document the exact error message that the Oracle test program displays.

9.2.      Examine the Oracle trace file produced when running the Oracle test program. The Oracle trace information is located in the *.TRC file. The Oracle trace file contains extended error information that is extremely helpful in diagnosing problems.

9.3.      Contact your Oracle support representative for assistance.

 

Compiling for the Oracle Test Program

You may compile the Oracle test program using the C++ compiler. The instructions above assume that you will name the resulting executable file MsDtcOra.exe.
