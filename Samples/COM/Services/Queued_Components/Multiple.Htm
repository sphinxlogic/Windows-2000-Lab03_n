Running the sample on multiple machines

In this step, we will distribute the queued component sample application over several machines. When an application calls a COM+ application on a remote machine, it does so via an application proxy. The steps below describe how to create and install an application proxy.

Running QC Sample #1

  1. On machine A, create and configure the COM+ application QCOrder per the instructions documented in Running the sample.
  2. On machine B, create and configure the COM+ application QCShip.
  3. On machine C, create and configure the COM+ application QCNotify.
  4. On machine B, export an application proxy for QCShip (see below).
  5. On machine C, export an application proxy for QCNotify.
  6. On machine A, import the application proxy created in step 4 (see below).
  7. On machine B, import the application export created in step 5.
  8. On machine A, run driver.exe.
  9. Experiment with running this configuration with QCShip and/or QCNotify not running.

Running QC Sample #2

  1. On machine A, create and configure the COM+ application QCOrder per the instructions documented in Running the sample.
  2. On machine B, create and configure the COM+ application QCShip.
  3. On machine C, create and configure the COM+ application QCNotify.
  4. On machine B, export an application proxy for QCShip2 (see below).
  5. On machine C, export an application proxy for QCNotify.
  6. On machine A, import the application proxies created in steps 4 & 5 (see below).
  7. On machine A, run driver.exe.
  8. Experiment with running this configuration with QCShip2 and/or QCNotify not running.

Exporting an application proxy

  1. If not already started, start the Component Services Explorer by selecting Start->Programs->Administrative Tools-> Component Services.
  2. Navigate to and highlight Console Root\Component Services Explorer\Computers\My Computer\COM+ Applications
  3. In the right pane you will see the icons for the COM+ applications, for example, QCShip.
  4. Right click on the application and select "Export...". This will start the COM Application Export Wizard
  5. Click Next.
  6. Enter the full path for the application file (.MSI file) you wish to create.
  7. Click the "Export as an Application Proxy" button.
  8. Click Next.
  9. Click Finish.
  10. This will create a .MSI file with the filename you specified. This file should be made available to the machine that is going to import and use the COM+ application.

Importing an application proxy

  1. If not already started, start the Component Services Explorer by selecting Start->Programs->Administrative Tools-> Component Services.
  2. Navigate to Console Root\Component Services Explorer\Computers\My Computer\COM+ Applications
  3. Right click on the COM+ Applications folder and select New->Application. This will start the COM Application Install Wizard.
  4. Click Next.
  5. Click the "Install pre-built application(s)" button.
  6. Navigate to and select the .MSI file created by exporting the application (see Creating a proxy application above).
  7. Click Open.
  8. Click Next.
  9. This will install the application into the directory shown.
  10. Click Next.
  11. Click Finish.
