Confirm the "Queued" property for the interfaces on the QCShip and QCNotify components

In this step you will confirm the "Queued" property for the interfaces on the following components:

o        QCShip

o        QCNotify

  1. If not already started, start the Component Services Explorer by pressing Start->Programs->Administrative Tools->Component Services.
  2. Navigate to and highlight Console Root\Component Services Explorer\Computers\My Computer\COM+ Applications\QCShip\Components\QCShip.Ship.1\Interfaces
  3. In the right pane you will see the icon for the IShip interface.
  4. Right click on IShip interface and select Properties.
  5. Select the Queuing tab.
  6. Confirm that the Queued option is checked.
  7. Repeat the step for INotify interface on the QCNotify.Notify.1 component in the QCNotify COM+ Application.

These interfaces were automatically set Queued during component install because the interfaces were marked QUEUEABLE in the IDL file used to define these interfaces. Refer to QCShip.idl and QCNotify.idl.
